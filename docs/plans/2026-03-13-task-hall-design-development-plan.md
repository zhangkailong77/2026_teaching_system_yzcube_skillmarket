# 任务广场设计开发计划（v1）

## 1. 目标
- 在任务端落地“任务广场”模块，支持学生/创作者高效发现任务并完成接单。
- 建立可扩展的任务列表能力：搜索、筛选、排序、分页、状态提示、接单动作。
- 保证与现有登录态、SSO 登录态兼容，未登录场景有明确引导。

## 2. 范围（v1）
- 页面：`/hall`（任务广场主页面）
- 能力：
  - 关键词搜索
  - 分类筛选（ComfyUI/短视频/智能体/平面设计）
  - 赏金区间筛选
  - 截止时间筛选
  - 最低能力分筛选
  - 排序（最新发布、赏金最高、即将截止）
  - 任务卡列表展示
  - 任务详情抽屉/弹层（先做轻量版）
  - “立即接单”动作（联通后端接口）
  - 收藏动作（先占位）

## 3. 非范围（v1 不做）
- 企业端发布审核流
- 任务推荐算法（个性化排序）
- 复杂协作（多人接单、议价、里程碑拆分）
- 站内消息中心深度联动

## 4. 用户与权限
- 游客：
  - 可浏览任务列表与详情
  - 点击“立即接单”时弹登录
- 已登录学生/创作者：
  - 可接单
  - 不满足门槛时展示明确拦截原因
- 企业账号：
  - v1 默认不在任务广场执行接单动作（后续按角色细分）

## 5. 信息架构与交互
### 5.1 页面结构
- 顶部：搜索栏 + 快捷排序
- 左侧（或顶部折叠）筛选区：类型、赏金、截止、能力分
- 主区域：任务卡网格（Masonry/Responsive Grid）
- 底部：分页或“加载更多”

### 5.2 任务卡字段（最小集）
- 任务标题
- 发布企业名
- 任务类型
- 赏金（积分）
- 截止时间（含剩余时间）
- 标签（如“急单”“高赏金”）
- 最低能力分要求
- 状态（可接单/已截止/已抢完/门槛不符）

### 5.3 关键状态
- `loading`：骨架屏
- `empty`：无数据提示 + 清空筛选按钮
- `error`：加载失败 + 重试按钮
- `ready`：正常展示

## 6. 后端接口规划（任务端）
### 6.1 列表查询
- `GET /api/v1/tasks`
- Query 参数：
  - `q`：关键词
  - `category`
  - `bounty_min` / `bounty_max`
  - `deadline_from` / `deadline_to`
  - `min_score`
  - `sort`（`latest` | `bounty_desc` | `deadline_asc`）
  - `page` / `page_size`
- 返回：
  - `items[]`
  - `total`
  - `page`
  - `page_size`

### 6.2 详情查询
- `GET /api/v1/tasks/{task_id}`

### 6.3 接单
- `POST /api/v1/tasks/{task_id}/claim`
- 校验：
  - 登录态有效
  - 任务可接
  - 用户未重复接单
  - 门槛满足
- 失败码建议：
  - `401` 未登录
  - `403` 门槛不满足
  - `409` 已被抢完/重复接单

## 7. 数据模型（v1 建议）
### 7.1 tasks
- `id`
- `title`
- `description`
- `category`
- `bounty_points`
- `deadline_at`
- `required_score`
- `status`（open/closed/full）
- `enterprise_name`
- `created_at`
- `updated_at`

### 7.2 task_claims
- `id`
- `task_id`
- `user_id`
- `status`（claimed/submitted/completed/cancelled）
- `created_at`
- `updated_at`

## 8. 前端实现计划
### Task 1：UI 结构重构
- 把当前 `/hall` 视图拆分为：筛选区、列表区、卡片组件、空态/错态组件。

### Task 2：状态与查询参数统一
- 建立 `hallQueryState`，并与 URL query 同步（可分享筛选结果链接）。

### Task 3：接入接口
- 新建 `tasks` service：
  - `fetchTasks`
  - `fetchTaskDetail`
  - `claimTask`

### Task 4：登录联动
- 未登录点击“立即接单”复用现有登录弹窗。
- 登录成功后自动继续执行上次接单动作（可选，v1 推荐实现）。

### Task 5：容错与反馈
- 加载、空态、失败重试、按钮 loading、防重复提交。

## 9. 验收标准（DoD）
- 能按条件筛选并返回正确结果。
- 接单链路可用：未登录拦截、登录后可接单、重复接单有提示。
- 页面在 `1920x1080` 与常见笔记本宽度下布局稳定。
- `npm run lint`、后端测试/最小冒烟通过。

## 10. 风险与对策
- 风险：任务状态并发更新导致“看到可接单但提交失败”。
- 对策：后端 `claim` 事务化校验 + 前端友好提示 `409`。

- 风险：筛选条件过多导致接口慢。
- 对策：索引设计（`category`、`deadline_at`、`status`、`bounty_points`）+ 分页默认值限制。

## 11. 里程碑
- M1（0.5 天）：确定字段与接口契约
- M2（1 天）：前端任务广场 UI + 列表查询联通
- M3（0.5 天）：接单接口联通 + 登录联动
- M4（0.5 天）：联调与回归、修复边界问题

## 12. 下一步

- 确认后进入实现：优先完成 Task 1~Task 3。
